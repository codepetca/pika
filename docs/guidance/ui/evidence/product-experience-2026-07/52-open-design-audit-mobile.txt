      - heading "1. Permanent deletion is too close to recovery" [level=3]
      - generic: Critical
      - paragraph:

      - paragraph: "Recommendation: Disable the legacy permanent Delete route and UI. Keep restore as the teacher action; any future hot-data removal must run only through verified compaction."

      - generic: High
      - paragraph:

      - paragraph: "Recommendation: Keep the same workflow states but expose one active student and one active grading region at a time on mobile, with persistent context and explicit next/previous controls."

      - heading "4. Dialog focus is managed, not contained" [level=3]
      - generic: High
      - paragraph:

      - paragraph: "Recommendation: Make one dialog primitive own focus entry, containment, Escape, background inertness, and trigger restoration; verify representative dialogs with keyboard and screen reader tests."
    - article:
      - heading "5. Dark borders are below non-text contrast" [level=3]
      - generic: High
      - paragraph:

    - article:
      - heading "6. Load failures can coexist with empty-state copy" [level=3]
      - generic: High
      - paragraph:
        - text: Observed · TeacherClassroomsIndex
        - text: An archived-list request failure sets an error, then the empty-state branch can still render “No archived classrooms.” That makes a system failure look like valid absence.
      - paragraph: "Recommendation: Make loading, error, empty, and loaded mutually exclusive states and keep retry attached to the error state."
    - article:
      - heading "7. Blueprint entry exposes structure before value" [level=3]
      - generic: High

      - heading "8. Archive and Gradex infrastructure outpace product surfaces" [level=3]
      - generic: High
